# Docker 容器网络模型

Docker 提供多种网络驱动，决定容器之间及容器与宿主机的通信方式。

## 三种主要模式

| 模式 | 通信范围 | 隔离性 | 典型场景 |
|---|---|---|---|
| bridge | 同宿主机容器间 | 有 | 单机开发、小规模部署 |
| host | 容器直接使用宿主机网络栈 | 无 | 高性能、端口密集场景 |
| overlay | 跨主机容器间 | 有 | Swarm / 集群部署 |

## bridge 模式工作原理

Docker 安装时自动创建 `docker0` 网桥（默认 172.17.0.0/16）。每个容器启动时：

1. 创建一对 veth pair（虚拟网卡对）
2. 一端放入容器命名空间，命名为 `eth0`
3. 另一端连接到 `docker0` 网桥
4. 容器通过网桥获得独立 IP，可与同网桥容器通信

端口映射（`-p 8080:80`）通过 iptables DNAT 规则实现。

## host 模式

使用 `--network=host` 时，容器不创建独立网络命名空间，直接使用宿主机的网卡和端口。

优点：无 NAT 开销，延迟最低。
缺点：失去网络隔离，多容器部署时端口冲突。

## overlay 模式

通过 VXLAN 隧道封装跨主机容器流量。需要 key-value 存储（如 Consul）或 Swarm 内置的 gossip 协议来协调。

适用场景：Docker Swarm 集群、跨主机微服务通信。

## 与其他容器运行时的对比

Podman 默认使用 CNI 插件而非 Docker 的 CNM，网络配置方式不同但底层实现类似。containerd 可以配合 CNI 插件使用，灵活性更高。
